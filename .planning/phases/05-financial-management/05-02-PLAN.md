# Plan: 05-02 Finances Filters & Sorting

**Phase:** 05 - Financial Management
**Plan:** 05-02
**Status:** Ready to execute

## Goal
Implement advanced filtering, sorting, and search capabilities for the transaction history to allow users to effectively analyze their finances.

## Context
- The current `FinanceCharts` and `FinancesPage` show all transactions without any filtering.
- Users need to be able to find transactions by date, category, and type.

## Task Breakdown
1. **Filter Bar**: Create a new component `FinanceFilterBar.tsx` with inputs for date range, category, and type.
   - Files: `src/features/finances/components/FinanceFilterBar.tsx`
2. **Search Hook Update**: Update `useTransactions` to accept filtering parameters and pass them to the API query.
   - Files: `src/features/finances/hooks/useTransactions.ts`, `src/features/finances/api/finances.api.ts`
3. **Sorting Implementation**: Add sorting options (Amount, Date) to the transaction list.
   - Files: `src/features/finances/components/FinanceCharts.tsx` (the transactions list part)
4. **Integration**: Add the filter bar to the top of the transactions list section in `FinancesPage.tsx`.
   - Files: `src/features/finances/pages/FinancesPage.tsx`

## Verification
- [ ] Changing the category filter updates the transaction list and charts correctly.
- [ ] Sorting by amount and date works in ascending and descending order.
- [ ] Filtering by date range accurately shows transactions within that period.
