# Plan: 10-02 Offline Experience

**Phase:** 10 - Performance Optimization & PWA
**Plan:** 10-02
**Status:** Ready to execute

## Goal
Provide a seamless experience even when the user is offline, including a dedicated fallback page and background synchronization for key actions.

## Context
- The app should not just show a generic "No Internet" browser page.
- Essential data (Habits, Tasks) should remain readable from cache.

## Task Breakdown
1. **Offline Page**: Create `src/app/pages/OfflinePage.tsx` with a premium brand-aligned design.
2. **Cache Strategy**: Configure `vite-plugin-pwa` to cache critical assets and the `/offline` route.
3. **Background Sync**: Implement background sync for Habits and Tasks so completed items are pushed to the server once the connection is restored.
   - Files: `src/shared/hooks/useRealtime.ts` (or service worker custom code).

## Verification
- [ ] Disabling network results in the custom Offline Page being shown (or cached app shell).
- [ ] App remains usable for viewing cached habits and tasks.
- [ ] Reconnecting syncs any pending changes automatically.
