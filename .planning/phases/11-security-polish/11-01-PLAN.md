# Plan: 11-01 Security Hardening

**Phase:** 11 - Security Hardening & Polish
**Plan:** 11-01
**Status:** Ready to execute

## Goal
Strengthen the application's security posture by implementing granular rate limits and ensuring rigorous input validation.

## Context
- The app currently has a global 100 req / 15 min limit.
- Brute-force attacks on Auth and cost-incurring AI calls need stricter limits.

## Task Breakdown
1. **Auth Rate Limiter**: Implement a strict limiter for `/api/auth/login` and `/api/auth/register` (e.g., 5 attempts per 15 minutes).
   - Files: `api/app.ts`
2. **AI Rate Limiter**: Implement a daily limit per user for AI features to prevent abuse.
   - Files: `api/app.ts`, `api/services/aiService.ts`.
3. **Validation Audit**: Ensure `zod` schemas in `api/routes/auth.ts`, `api/routes/finances.ts`, and `api/routes/tasks.ts` cover all required fields and correct data types.

## Verification
- [ ] Attempting to register multiple times in a row triggers a "Too many requests" error.
- [ ] Sending invalid JSON to any API route results in a clear validation error (400) without crashing the server.
