# Plan: 11-02 Accessibility & Design Polish

**Phase:** 11 - Security Hardening & Polish
**Plan:** 11-02
**Status:** Ready to execute

## Goal
Achieve high accessibility standards and ensure visual consistency across all modules of the application.

## Context
- The app uses a high-contrast dark theme (OLED), which helps with readability but requires careful attention to secondary text.
- Many icons in the Sidebar and feature cards lack descriptive labels for screen readers.

## Task Breakdown
1. **ARIA Audit**: Add `aria-label` to all NavLinks, icon buttons, and interactive elements.
   - Files: `src/app/layout/Sidebar.tsx`, `src/features/*/components/*`.
2. **Focus Management**: Ensure all modals (Transaction, Task, Onboarding) implement a focus trap and can be closed with the `ESC` key.
3. **Typography pass**: Standardize font-weights and sizes for headings to ensure a clear information hierarchy.
4. **Visual Consistency**: Audit border-radius and glass-morphism effects across all "Bento" cards.

## Verification
- [ ] Keyboard-only navigation allows using most features (Add task, log habit).
- [ ] Screen reader correctly identifies all navigation items.
- [ ] UI looks cohesive and follows the established design tokens.
