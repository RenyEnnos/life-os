# Plano 03-02: Funcionalidades Nativas e Bridge Tipada

## 🎯 Objetivo
Habilitar o poder do Desktop no Life OS, permitindo que a web-app interaja com o Sistema Operacional de forma segura e performática.

## 🧱 Must-Haves
- Ponte IPC tipada com TypeScript (ElectronAPI).
- Ícone na bandeja do sistema (Systray) com menu de contexto.
- Notificações nativas para lembretes de hábitos e tarefas.
- Persistência de estado da janela (tamanho e posição).

## 📋 Tarefas

<tasks>
- [x] **Definição de Tipos IPC**: Criar `src/shared/types/electron.ts` para definir a interface da ponte global.
- [x] **Implementação do Preload**: Expor as funções da bridge via `contextBridge`.
- [x] **Sistema de Notificações**: Conectar o sistema de mensagens do frontend com a API nativa de `Notification` do Electron.
- [x] **Systray Implementation**: Criar o ícone de bandeja com opções de "Abrir", "Esconder" e "Sair".
- [x] **Window State Persistence**: Integrar `electron-window-state` para que o app lembre onde foi fechado.
- [x] **Deep Link / Protocol Support**: Registrar o protocolo `lifeos://` para permitir abertura externa.
</tasks>
