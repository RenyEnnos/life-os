# Plano 02-01: Integridade Funcional do Dashboard e Widgets

## 🎯 Objetivo
Eliminar elementos não funcionais ("enfeites") do Dashboard, transformando o HealthWidget em uma ferramenta funcional e conectando as sugestões do Synapse a ações reais do sistema.

## 🧱 Must-Haves
- `HealthWidget` funcional com entrada manual de métricas básicas (Peso, Água, Sono) persistidas no Supabase.
- Botões "Executar" na `AgoraSection` disparando ações reais (ex: completar tarefa, logar hábito) baseado no tipo de sugestão.
- Remoção ou desativação explícita de qualquer elemento de UI que não possua backend funcional.

## 📋 Tarefas

<tasks>
- [ ] **Funcionalidade do HealthWidget**:
    - [ ] Criar schema `health.ts` em `src/shared/schemas`.
    - [ ] Criar `api/services/healthService.ts` e rotas correspondentes para persistência de métricas.
    - [ ] Substituir o placeholder do `HealthWidget.tsx` por um formulário de captura rápida ou exibição de dados reais.
- [ ] **Ações Reais no Synapse (AgoraSection)**:
    - [ ] Mapear `SynapseSuggestion` para tipos de ação conhecidos (`HABIT_LOG`, `TASK_COMPLETE`, `CALENDAR_EVENT`).
    - [ ] Implementar o dispatcher de ações no clique do botão "Executar".
- [ ] **Auditoria de Botões Fantasma**:

    - [ ] Verificar botões de "Add" nos Widgets e garantir que todos abrem o modal correto configurado na Fase 1.
</tasks>

- [ ] Nenhum console.log de "not implemented" ou handlers vazios restando nos componentes principais.
